Project description

Google App Engine is the Platform-as-a-Service (PaaS) platform for traditional web applications. It provide clean separation between computation and storage. This project demonstrates the various features of Google App Engine. It demonstrates the following concepts

  • Mega Store.
  • Data Model.
  • JPA.
  • Transactions.

Mega Store and Data Model

Mega Store is distributed storage system heavily rely on Big data table, Paxos and Chubby lock service for loosely coupled System Mega Store have strongly typed schema and data is presented by entities. The Primary key of Entity is different from Java EE Primary Key. Each entity is mapped to single big data row. It uses the hierarchical persistence model for each entity group. It is no sql data storage. It provide strong consistency within entity group where as loose consistency across entity group.

JPA

It provides object-relational mapping from RDBMS to NoSQL. It provide limited Support for Java Persistence API. It does not allow Join, Aggregation and polymorphic queries. It does not allow JPQL queries. All the queries are served by predefined index. We cannot use container manager entity manager with mega store. We have to manually create and close entity manager .

Transactions

It also limits the transaction support. There is no container managed transactions.Single transaction cannot create or operate on more than one root entity. There is no support for JTA in Google app Engine. Transactions does not contain any knowledge about entity group.
